# 💡 Як правильно формулювати запити для Deep Research

> Якість звіту Deep Research на 70% залежить від якості запиту. Ось перевірені прийоми.

---

## Структура ідеального запиту

```
[РОЛЬ] + [КОНТЕКСТ] + [ЗАВДАННЯ] + [ФОРМАТ] + [ОБМЕЖЕННЯ]
```

**Приклад:**
```
Ти — бізнес-аналітик. Я розробляю SaaS для автоматизації постингу 
у соцмережах для українського малого бізнесу.

Завдання: проаналізуй 5 найближчих конкурентів (Buffer, Hootsuite, 
Later, Publer, SmmPlanner).

Формат: порівняльна таблиця + розбір слабких місць кожного + 
висновок про незайняту нішу.

Фокус: ціни, відгуки реальних користувачів, підтримка українського 
ринку, інтеграції з Telegram.
```

---

## 10 прийомів для кращого результату

### 1. Вказуй свою роль і контекст
❌ *«Розкажи про ринок AI»*  
✅ *«Я підприємець, будую AI-сервіс для фрілансерів. Проаналізуй ринок AI-інструментів для фрілансу у 2026 році»*

---

### 2. Задавай конкретний часовий діапазон
❌ *«Які тренди у vector databases?»*  
✅ *«Які тренди у vector databases у 2025–2026 роках? Зосередься на змінах після появи GPT-4o і Llama 3»*

---

### 3. Проси порівняння, а не опис
❌ *«Що таке Qdrant?»*  
✅ *«Порівняй Qdrant, Weaviate і pgvector для production RAG-системи на 1M документів: швидкість, пам'ять, self-hosting, вартість»*

---

### 4. Вказуй бажаний формат виводу
- *«Зроби у форматі executive summary + детальний звіт»*
- *«Структуруй як: проблема → рішення → приклади → висновки»*
- *«Зроби таблицю порівняння, потім детальне пояснення кожного пункту»*
- *«Додай секцію TL;DR на початку»*

---

### 5. Обмежуй область пошуку
❌ *«Розкажи про автоматизацію»* (занадто широко)  
✅ *«Сфокусуйся тільки на no-code автоматизації (n8n, Make.com, Zapier) для малого бізнесу»*

---

### 6. Проси конкретні числа і джерела
*«Де можливо — наводь конкретні цифри: ринкові частки, ціни, показники продуктивності. Посилайся на первинні джерела»*

---

### 7. Ставитися критично — проси «інший бік»
*«Також включи критичні відгуки і відомі проблеми, не тільки переваги»*  
*«Яка критика цього підходу в академічній спільноті?»*

---

### 8. Використовуй вкладені запитання
Замість одного загального — структуруй підзавдання:
```
Дослідження повинно відповісти на 3 питання:
1. Який поточний стан ринку (розмір, гравці, тренди)?
2. Які незайняті ніші або больові точки клієнтів?
3. Яка оптимальна GTM-стратегія для нового гравця?
```

---

### 9. Вказуй цільову аудиторію звіту
*«Звіт для технічного засновника без маркетингового досвіду»*  
*«Пояснення для розробника, знайомого з Python але не з ML»*  
*«Для презентації інвестору без технічного бекграунду»*

---

### 10. Проси рекомендації та next steps
*«В кінці звіту додай конкретні рекомендації та 3–5 кроків, які я можу зробити наступного тижня»*

---

## Шаблони для різних задач

### Шаблон: Аналіз конкурентів
```
Зроби детальний competitive analysis для [ніша/продукт].
Цільова аудиторія: [хто]
Географія: [де]
Топ конкуренти (якщо знаєш): [список]

Включи: функції, ціни, сильні/слабкі сторони, відгуки, 
PR-скандали або проблеми, ринкову частку (якщо є дані).

Формат: таблиця + аналіз по кожному + загальні висновки.
```

---

### Шаблон: Технічне порівняння
```
Порівняй [технологія A] vs [технологія B] vs [технологія C].
Контекст використання: [твій кейс]
Критерії важливості (від найважливішого):
1. [критерій 1]
2. [критерій 2]
3. [критерій 3]

Результат: таблиця з оцінками + рекомендація для мого кейсу.
```

---

### Шаблон: Огляд стану ринку
```
Зроби market overview для [галузь/технологія] станом на 2026 рік.
Фокус:
- Розмір ринку та динаміка зростання
- Ключові гравці та їх частки
- Технологічні тренди наступних 2 років
- Регуляторне середовище (якщо релевантно)
- Можливості для нових стартапів

Джерела: переважно 2024–2026 рік, включи прогнози від аналітичних агентств.
```

---

## Типові помилки

| Помилка | Чому погано | Як виправити |
|---|---|---|
| Дуже короткий запит | Broad → поверхневий звіт | Додай контекст і критерії |
| Відсутність формату виводу | Звіт може бути незручно структурований | Вкажи структуру явно |
| Немає часового обмеження | Включить застарілі дані | Додай «за 2024–2026» |
| Один загальний запит замість підзапитань | Не охопить всі аспекти | Розбий на 3–5 конкретних питань |
| Не вказано цільова аудиторія | Рівень складності може не підійти | Вкажи свій бекграунд |

---

## Після отримання звіту

1. **Перевір джерела** — кликай на цитати, особливо для критичних рішень
2. **Задай уточнюючі питання** — «Розкажи детальніше про пункт 3»
3. **Попроси альтернативну думку** — «А які контраргументи?»
4. **Експортуй** — скопіюй Markdown або завантаж PDF для архіву
5. **Верифікуй ключові цифри** — статистику краще перевіряти в першоджерелах
