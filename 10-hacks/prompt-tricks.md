# Трюки з промптами для Perplexity

> Perplexity — не звичайний чатбот. Його промпти працюють інакше, бо він спершу шукає, а потім відповідає.

---

## 🔑 Ключова різниця від ChatGPT

Perplexity **спершу робить пошук**, а потім синтезує відповідь. Тому:
- Не треба просити "шукати в інтернеті" — він завжди шукає
- Варто формулювати запит як **пошукове питання**, а не як задачу для LLM
- Конкретика у промпті → кращі джерела → краща відповідь

---

## 🛠️ Трюк 1: Додавай контекст пошуку

❌ Погано:
```
Розкажи про n8n
```

✅ Добре:
```
Що нового в n8n у 2025 році? Останні оновлення, нові вузли, зміни в ціноутворенні.
```

Конкретна дата і тема → Perplexity знаходить актуальні джерела.

---

## 🛠️ Трюк 2: Формат відповіді

Perplexity добре розуміє інструкції щодо формату:

```
Поясни різницю між RAG і Fine-tuning.
Відповідь оформи у вигляді таблиці з колонками: критерій, RAG, Fine-tuning.
```

Інші корисні формати:
- `у вигляді нумерованого списку кроків`
- `коротко — до 5 речень`
- `як для розробника-початківця`
- `з прикладами коду на Python`

---

## 🛠️ Трюк 3: Вказуй джерело явно

```
Знайди на Reddit думки про Cursor IDE у 2025 році
```

```
Що пишуть на Hacker News про Perplexity API?
```

Perplexity розуміє посилання на конкретні платформи і намагається шукати там (через Focus або органічно).

---

## 🛠️ Трюк 4: Уточнення через follow-up

Perplexity зберігає контекст у межах сесії:

```
Запит 1: Порівняй Supabase і Firebase
Запит 2: Яка з них краща для real-time чату?
Запит 3: Покажи приклад коду для підписки на оновлення в Supabase
```

Кожен follow-up уточнює — не починай нову сесію без потреби.

---

## 🛠️ Трюк 5: Промпт для перевірки фактів

```
Чи правда, що [твердження]? Знайди підтвердження або спростування з першоджерел.
```

Perplexity відмінно справляється з fact-checking, бо цитує джерела.

---

## 🛠️ Трюк 6: Системний промпт через Spaces

Якщо маєш Pro — у Spaces можна задати **системний промпт** для всіх запитів у цьому просторі:

```
Ти — технічний асистент для розробника на Python та n8n.
Відповідай українською мовою.
Включай приклади коду там, де це можливо.
Цитуй офіційну документацію, коли вона доступна.
```

---

## 📋 Шаблони промптів

### Ресерч конкурентів
```
Зроби огляд [компанія/продукт] у 2025 році:
- Ключові фічі
- Ціни
- Відгуки користувачів (Reddit, G2, Product Hunt)
- Порівняй з [конкурент]
```

### Технічний огляд
```
Поясни як працює [технологія]:
- Архітектура (коротко)
- Переваги та недоліки
- Коли використовувати
- Приклад на Python
```

### Новини теми
```
Що нового в [тема] за останні 30 днів?
Фокус на: практичні зміни, нові релізи, важливі анонси.
```
