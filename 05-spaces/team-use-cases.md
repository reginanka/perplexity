# 👥 Spaces для команд і бізнесу — реальні кейси

> Конкретні сценарії використання Perplexity Spaces у командній роботі та бізнес-процесах.

---

## Кейс 1: База знань для онбордингу

**Проблема:** Кожен новий співробітник витрачає тижні на вивчення внутрішніх процесів.

**Рішення:**
```
Space: "Онбординг нових співробітників"
├── company-handbook.pdf
├── processes-guide.docx
├── tech-stack-overview.md
├── faq-new-employees.pdf
└── 🤖 Промпт: "Ти — HR-асистент. Відповідай дружньо, посилайся на конкретні розділи документів."
```

**Результат:** Новачок ставить питання напряму в Space і отримує точну відповідь з посиланням на внутрішній документ, а не чекає відповіді від HR-менеджера.

**Економія:** ~5 годин HR-менеджера на кожного нового співробітника.

---

## Кейс 2: Конкурентна розвідка

**Проблема:** Команда маркетингу не знає актуальних позицій конкурентів.

**Рішення:**
```
Space: "Аналіз конкурентів Q2 2025"
├── competitor-A-analysis.pdf
├── competitor-B-pricing.docx
├── 🔗 https://competitor-a.com
├── 🔗 https://competitor-b.com/pricing
└── 🤖 Промпт: "Аналізуй наших конкурентів об'єктивно. Шукай у файлах + свіжі дані з веб."
```

**Що можна питати:**
- _"Які фічі є у Competitor A, яких немає у нас?"_
- _"Порівняй ціни на enterprise план з нашими"_
- _"Що змінилось у Competitor B за останній місяць?"_ (веб-пошук)

---

## Кейс 3: Підтримка клієнтів / FAQ

**Проблема:** Команда підтримки постійно відповідає на одні й ті самі питання.

**Рішення:**
```
Space: "Customer Support KB"
├── product-faq.pdf
├── troubleshooting-guide.docx
├── refund-policy.pdf
├── changelog-2025.md
└── 🤖 Промпт: "Відповідай тільки на основі документів. Якщо відповіді немає — скажи 'потребує уточнення у команді'."
```

**Для кого:** Агенти підтримки можуть швидко знаходити відповіді замість пошуку по Confluence/Notion.

> ⚠️ Не підходить для публічного бота — немає API. Для публічного FAQ потрібен Perplexity API або власний RAG.

## Кейс 4: Особистий ресерч-простір

**Для:** Freelancer, дослідник, підприємець.

```
Space: "AI Automation Research 2025"
├── n8n-documentation.pdf
├── make-com-best-practices.docx
├── 🔗 https://docs.n8n.io
├── saved-articles/*.pdf (10+ статей)
└── 🤖 Промпт: "Ти — мій дослідницький асистент з AI-автоматизації. Відповідай технічно, зі прикладами коду."
```

**Використання:**
- Зберігати важливі статті та документацію
- Ставити складні технічні питання по своїх матеріалах
- Порівнювати підходи з різних джерел одночасно

---

## Кейс 5: Юридична / Compliance база

**Для:** Юридичні відділи, фінтех, медтех.

```
Space: "Legal & Compliance"
├── gdpr-guidelines.pdf
├── company-contracts-templates.docx
├── internal-compliance-checklist.pdf
└── 🤖 Промпт: "Відповідай точно і обережно. Завжди додавай: 'це інформаційна довідка, не юридична консультація'."
```

**Економія:** Менше звернень до юристів за базовими питаннями.

---

## Кейс 6: Підготовка до зустрічей

**Швидкий кейс для менеджерів:**

```
Space: "Client: Acme Corp"
├── acme-contract.pdf
├── meeting-notes-history.docx
├── acme-requirements.pdf
└── 🤖 Промпт: "Ти — помічник для роботи з клієнтом Acme Corp. Знай їх контекст напам'ять."
```

Перед кожною зустріччю: _"Що ми обговорювали минулого разу і які були відкриті питання?"_

## Порівняння Spaces з альтернативами для командної роботи

| Інструмент | Сильні сторони | Слабкі сторони |
|-----------|---------------|----------------|
| Perplexity Spaces | Швидко, AI-відповіді, веб+файли | Немає API, обмежений обсяг |
| Notion AI | Інтегровано з нотатками | Дорого ($16+/user), слабкий RAG |
| Confluence + AI | Корпоративний стандарт | Складно, дорого, повільно |
| Flowise/RAG | Повний контроль, API | Потрібна технічна команда |
| ChatGPT Projects | Простий UI | Немає веб-пошуку в контексті файлів |

---

## Поради для команд

1. **Один Space = одна ціль** — не змішуй HR, маркетинг і підтримку в одному Space
2. **Промпт — це конституція Space** — витрать 10 хвилин на якісний промпт
3. **Оновлюй файли регулярно** — застарілі документи дають неточні відповіді
4. **Тестуй з командою** — нехай кожен задасть по 3 тестових питання після налаштування
5. **Документуй ліміти** — скажи команді, що Spaces не замінює CRM чи Jira
